# 마인드 피트니스 면역공방 예약 웹페이지

**버전**: 1.3.0  
**최종 업데이트**: 2024-11-26  
**변경 이력**: 
- v1.0.0: 초기 버전 (카카오맵/Google Maps 사용)
- v1.1.0: react-leaflet + OpenStreetMap으로 변경 (API 키 불필요)

- v1.3.0: 노션 DB "이체확인" 속성 제거

## 1. Overview

### Problem Statement

* **문제 정의**:
  제주도 마인드 피트니스 회원들이 *면역공방*에서 제공하는 면역 스파/처치를 받고 싶어도,
  현재는 오프라인/전화/구두로만 예약이 이뤄져 번거롭고,
  “어떤 서비스인지, 어떻게 예약하는지, 얼마인지” 한 번에 보이도록 안내해주는 온라인 창구가 없다.

* **왜 중요한가**:

  * 기존 회원에게 새로운 면역 프로그램을 자연스럽게 소개할 수 있는 중요한 접점이다.
  * 첫 방문 50% 할인이라는 좋은 오퍼가 있어도, 체계적인 예약 동선이 없으면 전환율이 떨어진다.
  * 운영 입장에서도 예약 현황을 정리하고, 고객 정보를 미리 받아두면 현장 운영이 훨씬 수월해진다.

* **타겟 고객**:

  * **대상**: “제주 마인드 피트니스”를 이미 이용하고 있는 기존 회원 전원
  * **특징**:

    * 건강/체력 관리에 관심이 많은 사람들
    * 새로운 웰니스·면역 프로그램에 관심이 높음
    * 모바일로 간단히 예약하고 싶어하는 20–50대 중심

* **전략적 적합성**:

  * 마인드 피트니스 기존 회원 → 면역공방 서비스로 자연스럽게 확장시키는 **크로스셀 채널**이 된다.
  * 향후 다른 프로그램(예: PT, 명상 클래스, 워크숍 등) 예약 페이지로도 구조를 재사용할 수 있는 **기본 예약 템플릿**이 된다.
  * “오프라인 중심 센터 → 온라인 예약/상담 채널”로 확장하는 첫 단계로, 브랜드의 디지털 전환 방향성과도 맞는다.

* **핵심 전제/가정**:

  * 대상 사용자는 이미 마인드 피트니스 회원이므로, 로그인 없이 이름/연락처만으로도 본인 식별이 가능하다.
  * 첫 방문 50% 할인은 강력한 예약 유인으로 작동할 것이다.
  * 별도의 복잡한 회원 시스템 없이도, 단일 웹페이지와 간단한 예약 폼만으로도 충분히 MVP를 검증할 수 있다.
  * 계좌 입금 방식(선입금/예약금)은 현재 센터의 운영방식과 잘 맞고, 사용자에게도 크게 낯설지 않다.

### Proposed Work

* **솔루션 개요**:
  마인드 피트니스 기존 회원이 *면역공방*의 면역 스파/처치를 간단히 이해하고,
  웹페이지 한 곳에서 **첫 방문 50% 할인 안내 + 예약 + 계좌 입금 안내 + 예약 변경**까지 할 수 있는 “단일 예약 랜딩 페이지”를 만든다.

* **핵심 기능 (3–5개)**:

  1. **면역공방 서비스 소개 섹션**

     * 상단 히어로 영역에 “면역 스파/처치를 받으면 무엇이 좋은지”를 센스 있는 2–3줄 문장으로 요약
     * 첫 방문 50% 할인 안내, 위치(제주시 해안가 인근) 기본 정보 제공
  2. **예약하기 버튼 & 예약 폼**

     * 메인 화면에서 바로 보이는 “예약하기” 버튼
     * 클릭 시 이름 / 이메일 / 전화번호 / 희망 날짜 / 희망 시간 입력 가능한 폼으로 이동
  3. **예약 완료 & 계좌번호 안내 섹션**

     * 예약 폼 제출 후, 예약 접수 완료 메시지
     * 계좌번호, 예금주, 입금 안내 문구(예: “예약 후 24시간 이내 입금 시 확정”) 표시
  4. **예약 변경 진입 버튼 (우측 상단)**

     * 웹페이지 우측 상단에 “예약 변경” 버튼 고정
     * 클릭 시 이름 + 전화번호를 입력하는 간단한 인증 폼으로 이동
  5. **예약 변경 페이지**

     * 이름/전화번호로 기존 예약 정보 조회
     * 희망 날짜/시간 수정 또는 취소 요청 가능(실제 로직은 이후 단계에서 구현/운영 방식 정의)
  6.   **면약공방 위치표시**

     * 화면 가장 하단에 면역공방 주소(제주 제주시 도두항서7길 2 2층)와 그 위치 지도표시
* **사용자 플로우**: 사용자가 제품을 어떻게 사용하는가?

  **A. 신규 예약 플로우**

  1. **[메인 랜딩 페이지] → [스크롤 없이 상단 히어로 영역 확인] → [면역공방 소개 2–3줄 문장 및 첫 방문 50% 할인 안내 인지]**
  2. **[메인 랜딩 페이지] → [“예약하기” 버튼 클릭] → [예약 폼 페이지/섹션으로 이동]**
  3. **[예약 폼 화면] → [이름, 이메일, 전화번호, 희망 날짜, 희망 시간 입력] → [‘예약 완료’ 버튼 클릭]**
  4. **[예약 완료 화면] → [“예약 접수 완료” 메시지 확인] → [계좌번호, 예금주, 입금 기한 및 입금 시 유의사항 확인]**
  5. **[사용자] → [본인 은행 앱/계좌에서 입금 진행] → [센터 측은 입금 확인 후 별도 문자/전화로 확정 안내 (운영 프로세스)]**

  **B. 예약 변경 플로우**

  1. **[어느 화면이든] → [우측 상단 “예약 변경” 버튼 클릭] → [예약 변경 전용 페이지/모달로 이동]**
  2. **[예약 변경 화면] → [이름 + 전화번호 입력] → [‘예약 조회’ 버튼 클릭]**
  3. **[예약 정보 화면] → [기존 예약 일시 확인] → [새로운 날짜/시간 선택 후 수정] 또는 [예약 취소 요청]**
  4. **[예약 변경 완료 화면] → [변경/취소 결과 메시지 표시] → [필요 시, 안내 문구 (예: 변경 가능 시점, 취소 수수료 정책 등) 노출]**

## 2. Decision / Risk

### Stakeholders

* **직접 이해관계자**:

  * 마인드 피트니스 회원(면역공방 예약 이용자)
  * 면역공방 운영 담당자(예약을 실제로 보고 관리하는 사람)
  * 마인드 피트니스/면역공방 원장 또는 운영 책임자
* **간접 이해관계자**:

  * 센터 리셉션/카운터 직원(예약·입금 확인·안내 담당)
  * 마케팅 담당자(웹페이지 링크를 회원에게 공유하는 역할 – 문자, 카톡 등)

---

### Risk

* **주요 위험 요소 & 발생 가능성과 영향도 & 대응 방안**:

  1. **예약은 했지만 입금을 안 하는 경우**

     * 발생 가능성: 중간 / 영향도: 높음
     * 대응 방안:

       * 예약 완료 페이지에 “입금 기한(예: 24시간) 이후 자동 취소”를 명확히 고지
       * 추후에는 입금 여부 필터링을 위해 간단한 관리용 스프레드시트/DB와 연동 고려
  2. **예약 변경 기능의 기술적 복잡성** (기존 예약 조회/수정 로직)

     * 발생 가능성: 중간 / 영향도: 중간
     * 대응 방안:

       * MVP 1차에서는 “이름 + 전화번호 + 변경 희망 시간 → 변경 요청 접수” 수준으로 단순화, 실제 변경은 운영자가 수동 반영
       * 이후 예약 조회/수정 자동화를 v1.1 또는 v2에서 구현
  3. **사용자가 예약 완료 후 계좌번호를 잊어버리는 경우**

     * 발생 가능성: 중간 / 영향도: 낮음~중간
     * 대응 방안:

       * 예약 완료 후 “문자 또는 카카오 알림톡으로 계좌번호 재안내” 프로세스를 별도 운영
       * 예약 완료 화면에 “스크린샷 저장을 권장합니다” 같은 안내 추가
  4. **비회원이 링크를 보고 예약하려는 경우**

     * 발생 가능성: 낮음 / 영향도: 낮음
     * 대응 방안:

       * 페이지에 “마인드 피트니스 회원 전용 예약 페이지” 문구 명시
       * 비회원의 경우, 센터로 문의하도록 문구 추가

## 3. Success Criteria

### 핵심 지표 (2–3개)

1. **지표명**: 예약 폼 제출 수

   * **측정 방법**: 웹페이지 내 예약 폼 제출 횟수(월간 기준)
   * **목표 수치**: 런칭 후 1개월 내 신규 예약 30건 이상
   * **달성 기한**: 런칭 후 1개월

2. **지표명**: 예약 → 실제 입금 전환율

   * **측정 방법**: 예약 완료 건 중, 실제 계좌 입금이 확인된 비율
   * **목표 수치**: 60% 이상
   * **달성 기한**: 런칭 후 1–2개월

3. **지표명**: 예약 변경 기능 사용률(선택)

   * **측정 방법**: 전체 예약 대비 “예약 변경” 버튼을 통해 변경/취소 요청을 한 비율
   * **목표 수치**: 20% 이하(예약 변경이 적다는 것은 일정 만족도가 높은 것으로 간접 해석 가능)
   * **달성 기한**: 런칭 후 2개월

### 보조 지표

* 랜딩 페이지 방문 수(월간)
* 방문 대비 예약 버튼 클릭률(CTR)
* 예약 완료 후 이탈(입금 미완료) 사유(운영 측에서 수집하는 정성 데이터)

## 4. Definition of Done

### 필수 조건 (Done)

* 메인 랜딩 페이지에서 **면역공방 소개 문구(2–3줄)**와 **첫 방문 50% 할인 안내**가 명확히 보인다.
* “예약하기” 버튼을 클릭하면 **이름, 이메일, 전화번호, 희망 날짜, 희망 시간**을 입력할 수 있는 예약 폼이 정상적으로 동작한다.
* 예약 폼 제출 시, **예약 정보가 저장되거나 이메일/시트로 전달**되어 운영자가 확인할 수 있다.
* 예약 완료 화면에 **계좌번호, 예금주, 입금 안내 문구**가 표시된다.
* 우측 상단에 “예약 변경” 버튼이 항상 노출된다.
* “예약 변경” 버튼 클릭 → 이름+전화번호 입력 → 예약 변경 요청이 가능한 화면까지의 플로우가 완성된다.

### 개선 필요 (but 출시 가능)

* 예약 변경이 “자동 수정”이 아니라, 우선은 “변경 요청 접수” 형식으로 운영자가 수동 처리하는 구조
* 반응형 디자인(모바일/PC 최적화)이 100% 완벽하지 않더라도 기본 사용에는 문제가 없는 수준
* 이메일/문자 자동 발송 기능이 없고, 우선은 예약 정보만 수집하는 형태

### 치명적 (출시 불가)

* 예약 폼이 제출되지 않거나, 정보가 전혀 저장/전달되지 않는 경우
* 계좌번호 정보가 잘못 표기되어 입금 오류가 발생할 위험이 있는 경우
* “예약하기” 버튼 또는 “예약 변경” 버튼이 모바일 환경에서 클릭 불가능한 경우
* 첫 방문 50% 할인 정책과 실제 청구 금액이 계속 불일치하는 경우

## 5. Nope (하지 않을 것)

1. **기능명**: 회원 가입/로그인 시스템

   * **이유**: 대상은 이미 ‘마인드 피트니스 회원’이며, 예약 목적에 비해 개발 복잡도가 크다.
   * **대안**: 이름 + 전화번호 기반 간단 식별, 필요 시 운영자가 수동으로 회원 여부 확인
   * **재고려 시점**: 예약 볼륨이 크게 늘어나고, 다른 서비스(PT, 클래스 등)까지 통합 관리할 때

2. **기능명**: 실시간 잔여 슬롯(타임 슬롯별 자동 마감 표시)

   * **이유**: 초기 예약 볼륨과 운영 방식 상, 복잡한 캘린더/슬롯 시스템은 오버스펙
   * **대안**: 운영자가 수동으로 가능한 시간대를 관리하고, 예약이 겹칠 경우 개별 연락으로 조정
   * **재고려 시점**: 일일 예약 건수 증가 및 자동화 필요성이 높아졌을 때

3. **기능명**: 온라인 결제(카드결제, PG사 연동)

   * **이유**: PG 연동, 수수료, 정산 등 초기 셋업 비용과 시간이 크다.
   * **대안**: 계좌 입금 방식 유지 + 추후 카드 결제 필요성이 명확해질 때 고려
   * **재고려 시점**: 월 예약/매출 규모가 일정 수준 이상이 되거나, 고객 요청이 반복될 때

## 6. Post-Launch Plan

### v1.1 (Quick Wins)

* 예약 완료 시, 운영자에게 자동 이메일 발송 기능 추가
* 예약 완료 화면에 “카카오톡 공유하기/링크 복사하기” 버튼 추가(지인 추천용)
* 예약 변경 요청 시, 운영자에게도 알림(이메일/슬랙 등) 전달

### v2 (Major Features)

* 간단한 **관리자용 예약 리스트 페이지** (날짜별 예약 현황 확인)
* 기본적인 통계 화면: 월별 예약 수, 첫 방문 고객 비율 등
* 예약 변경 로직을 반자동/자동으로 업그레이드(시간 중복 체크 등)

### v3+ (Vision)

* 마인드 피트니스 전체 프로그램(PT, 그룹 클래스, 워크숍 등)에 공통적으로 적용 가능한 **통합 예약 플랫폼**으로 확장
* 회원 전용 마이페이지(내 예약 내역, 과거 이용 이력, 추천 프로그램)
* 온라인 결제/정기 결제 연동
* 마케팅 자동화(첫 방문 후 N일 뒤 재방문 쿠폰 발송 등)
